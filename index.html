AI → ML → Deep Learning → GenAI Hierarchy
AI is the broadest field (machines behaving intelligently). ML is a subset that learns from data. Deep Learning is a subset of ML using neural networks. GenAI is a subset of deep learning that generates new content.
Exam trigger Any question comparing these four terms — draw the nesting: AI > ML > Deep Learning > GenAI
ESOL note

Think of Russian nesting dolls. The biggest doll is AI (everything). Inside is ML. Inside that is Deep Learning. Inside that is GenAI. Every inner doll is a type of the outer one — but not vice versa.

Supervised / Unsupervised / Reinforcement Learning
Supervised: labelled data, learns to predict (classification → category; regression → number). Unsupervised: unlabelled data, finds patterns (clustering). Reinforcement: learns by reward/penalty through trial and error.
Exam trigger "What type of ML is being used?" — ask yourself: Is the data labelled? Yes → Supervised. No → Unsupervised. Is it learning by doing? → Reinforcement.
ESOL note

Supervised = teacher gives answers. Unsupervised = student groups things alone. Reinforcement = player learns a game by winning/losing points.

ML Pipeline — All 9 Stages
1. Data Collection → 2. EDA (explore patterns) → 3. Pre-processing (clean/format) → 4. Feature Engineering (select inputs) → 5. Model Training → 6. Hyperparameter Tuning → 7. Evaluation → 8. Deployment → 9. Monitoring
Exam trigger Ordering/matching questions: put stages in the right sequence. Remember: you cannot train before collecting data; you cannot deploy before evaluating.

Model Evaluation Metrics (Performance + Business)
Technical: Accuracy (% correct), F1 Score (precision + recall balance — best for imbalanced data), AUC-ROC (separation between classes, 0–1). Business: ROI, cost per user, development costs, customer feedback.
Exam trigger "Which metric is most appropriate?" — Imbalanced dataset (fraud, disease) → F1 or AUC. Balanced dataset → Accuracy. Business impact question → ROI or cost per user.
When to Use vs When NOT to Use AI/ML
Use AI when: decision-making at scale, automation of complex patterns, scalability beyond human capacity. Do NOT use AI when: a simple rule works (tax = price × 0.2), specific guaranteed outcome needed, insufficient data, cost outweighs benefit.
Exam trigger "Should this company use AI?" — If the answer can be computed with a formula or lookup table, the answer is NO. If the pattern is complex and the dataset is large, the answer is YES.

Bias-Variance Tradeoff — Full Mechanics
High bias = underfitting (model too simple — wrong in the same direction every time). Fix: more features, more complex model, more training. High variance = overfitting (model too complex — memorises noise). Fix: more data, regularisation, simpler model, dropout. The tradeoff: making a model more complex reduces bias but increases variance. Making it simpler reduces variance but increases bias. Goal: find the sweet spot.
Exam trigger "Model underfits" → increase complexity. "Model overfits" → reduce complexity or add data. "Cannot eliminate both" → bias-variance tradeoff.
ESOL note

Imagine a student: high bias = always gets the wrong answer in the same way (too simple a strategy). High variance = gets wildly different answers each time depending on which questions they studied (memorised instead of understood). The ideal student gives consistent, correct answers on unseen questions.

Train / Validation / Test Split and Data Leakage
Training set: model learns from this. Validation set: tune hyperparameters and monitor overfitting during development. Test set: final evaluation — never seen during training or tuning. Data leakage: when information from outside the training set contaminates the model — causes falsely high accuracy. Example: including the target variable as an input feature.
Exam trigger "Model performs well during development but fails in production" → data leakage. "Evaluate final model performance" → test set. "Tune hyperparameters" → validation set.
Example

A fraud model trained on data that included the fraud investigation outcome (a post-event feature) appears 99% accurate but fails in production because that feature is never available at prediction time — classic data leakage.

Neural Networks & Deep Learning
Neural networks are computing systems inspired by the brain — layers of interconnected nodes that process data. Deep learning uses many layers (hence "deep") to learn complex features automatically.

Data Types in AI
Structured: rows & columns (databases, spreadsheets). Unstructured: text, images, audio, video. Labelled: has known correct answer. Unlabelled: no answer provided. Tabular: table format. Time-series: ordered by time (stock prices, sensor data).
Exam trigger Identify data type from scenario description. Text/images/audio = unstructured. Rows & columns = structured/tabular. Data with timestamps = time-series.
Classification vs Regression
Classification: output is a category (spam/not spam, churn yes/no). Regression: output is a continuous number (house price, sales forecast).
Exam trigger Category answer → Classification. Number answer → Regression. "Which ML technique?" — look at the output type first.
Batch vs Real-time Inferencing
Batch: runs predictions on large datasets all at once — high throughput, delayed results. Real-time: instant prediction on a single input — low latency, immediate response.
Exam trigger Time-sensitive action (fraud check at checkout, instant loan approval) → Real-time. Overnight processing (monthly reports, bulk scoring) → Batch.

GenAI Advantages vs Disadvantages
Advantages: Adaptability (works across many tasks), Responsiveness (answers in real time), Simplicity (no ML expertise needed to use). Disadvantages: Hallucinations (confident wrong answers), Interpretability (hard to explain decisions), Inaccuracy (not always correct), Nondeterminism (different output each time).
Exam trigger "Which is a benefit/limitation of GenAI?" — match the exam option to the official terms: adaptability, responsiveness, simplicity vs hallucination, interpretability, inaccuracy, nondeterminism.
ESOL note

A very knowledgeable but unreliable assistant: great at many things quickly (advantage) but sometimes makes things up and you cannot always tell when (disadvantage). These are the trade-offs of GenAI.

Token / Chunking / Embedding / Vector — Core GenAI Terms
Token: smallest unit processed (~0.75 words). Context window: max tokens in one request. Chunking: splitting large docs into smaller pieces for RAG. Embedding: list of numbers representing meaning. Vector: the numerical representation used for similarity search.
Exam trigger Questions about "how does RAG store documents?" → chunking + embeddings + vectors. "Why does the model forget earlier conversation?" → context window limit.
ESOL note

Token = puzzle piece of a word. Context window = the table you can spread pieces on (limited space). Chunking = cutting a big document into smaller puzzle sets. Embedding = a map coordinate for meaning. Vector = those coordinates as numbers.

Cost Tradeoffs: Token Pricing vs Provisioned Throughput
Token-based pricing: pay per token processed (input + output). Best for variable, unpredictable workloads. Provisioned throughput: reserve dedicated model capacity — fixed cost, guaranteed performance. Best for high-volume, consistent workloads.
Exam trigger "Which pricing model?" — Variable usage → token-based. High consistent volume needing guaranteed speed → provisioned throughput.
ESOL note

Token pricing = pay-as-you-go mobile plan (pay for exactly what you use). Provisioned throughput = monthly unlimited plan (fixed price, guaranteed capacity). Choose unlimited when you use it heavily every day; pay-as-you-go when usage is unpredictable.

Customer Lifetime Value (CLV): total revenue from a customer over time. Accuracy: % of correct outputs. Cross-domain performance: how well the model works across different subject areas.

Transformer Architecture
The neural network design powering modern LLMs. Uses a mechanism called self-attention to understand relationships between all words in a sequence simultaneously — not one word at a time. This makes it much faster and more accurate than older RNN approaches.
Exam trigger Questions about "what architecture powers LLMs?" or "what makes modern AI understand context?" → Transformer / self-attention.
Multimodal Models
AI models that process and generate multiple types of data — text, images, audio, and video — within a single model. They understand how different modalities relate to each other.
Exam trigger "Analyse an image and explain it in text" or "generate a video from a text description" → multimodal model.
Diffusion Models
A type of generative model used for image, audio, and video generation. Learns by adding noise to data progressively, then learning to reverse the process — removing noise to generate new content.
Exam trigger "AI image generation" or "generate realistic images from text" → diffusion model. Amazon Titan Image Generator uses diffusion.
Prompt Engineering Techniques (All 5)
Zero-shot: no examples. Single-shot / One-shot: exactly 1 example. Few-shot: 2–5 examples. Chain-of-thought: "think step by step" — improves reasoning. Negative prompt: tells the model what NOT to include.
Exam trigger Ordering/matching: count the examples in the prompt. 0 → zero-shot. 1 → single-shot. 2–5 → few-shot. "Think step by step" instruction → chain-of-thought.

RAG Architecture — How it Works
1. User submits a query. 2. Query is converted to an embedding. 3. Embedding is compared to document embeddings in a vector DB. 4. Most similar chunks retrieved. 5. Retrieved chunks + original query injected into the FM prompt. 6. FM generates grounded answer.
Exam trigger "Reduce hallucination using company documents" → RAG. "Keep AI answers up to date without retraining" → RAG. Always involves: vector DB + embedding + retrieval + FM.
ESOL note

Open-book exam: instead of relying on memory alone, the AI looks up relevant notes first, then writes the answer based on what it found. The vector DB is the notebook, embeddings are the index, retrieval finds the right page.

FM-Specific Explainability Mechanisms
Traditional ML uses Shapley values and PDPs. Foundation models have different explainability tools: Attention weights: show which input tokens the model focused on when generating each output — the primary FM explainability mechanism. Feature attribution: scores showing which input features most influenced the FM output. Prompt transparency: a well-designed, documented system prompt explains the decision criteria baked into the FM. Post-hoc explainability: techniques applied AFTER the model produces output — SHAP values, LIME, attention maps. Chain-of-thought outputs: prompting the FM to show its reasoning before the final answer.
Exam trigger "Explain FM decision" / "audit FM output" / "which inputs influenced the FM" → attention weights or feature attribution. "Document FM decision criteria" → system prompt transparency. "Explain after the fact" → post-hoc tools.
Example

A bank uses an FM for loan decisions. Regulators ask why a specific loan was denied. The team uses attention weights to show the model focused heavily on the debt-to-income ratio field, and chain-of-thought prompting to produce a human-readable reasoning trace.

Transfer Learning
Using a model already trained on one task as the starting point for a different but related task. The model transfers its learned knowledge rather than starting from scratch — saving compute and data.
Exam trigger "Adapt a model trained on general text to medical records without full retraining" → Transfer learning.

Partial Dependence Plot (PDP)
A visualisation showing how a single input feature affects model predictions, holding all other features constant. Answers: "if I change just this one variable, how does the prediction change?" Used to explain model behaviour to regulators, auditors, and business stakeholders. Generated by SageMaker Clarify.
Exam trigger "Show how one feature affects predictions" / "explain to regulators which variable drives decisions" / "visualise model behaviour for a single input" → PDP.

Dataset Characteristics for Responsible AI
Inclusivity: represents all relevant groups (age, gender, ethnicity, language). Diversity: wide variety of examples and perspectives. Curated data sources: carefully selected, vetted sources of known quality. Balanced datasets: roughly equal representation of all classes/groups.
Exam trigger "What makes a dataset responsible?" → these four characteristics. Imbalanced dataset = fairness risk. Non-curated = quality risk. Single demographic = inclusivity risk.
Bias Effects: Variance, Overfitting, Underfitting
High variance (overfitting): model memorises training data, fails on new data — results in inaccuracy for unseen groups. High bias (underfitting): model too simple, misses real patterns — systematically wrong across all groups. Effect on demographics: biased training data causes the model to perform better for over-represented groups and worse for under-represented ones.
Exam trigger "A model performs well for group A but poorly for group B" → likely data bias or model bias affecting that demographic subgroup.
Open Source Models & Licensing as Transparency Tools
Using open-source foundation models (Llama, Mistral) increases transparency because the architecture, training methodology, and weights are publicly available for inspection. Licensing terms define what data was used and how the model can be used commercially.
Exam trigger "How can a company increase model transparency?" → open-source models. "Verify what data was used to train the model" → licensing documentation / model cards.
Human-Centered Design for Explainable AI
Designing AI systems so that explanations are useful and understandable to the specific humans who use or are affected by the system. Includes: designing explanations for the audience's literacy level, testing whether explanations actually help decision-making, and involving end users in the design process.
Exam trigger "Design AI explanations that non-technical users can understand and act on" → human-centered design for XAI.
Interpretability vs Performance Tradeoff
Simple models (linear regression, decision trees): high interpretability, lower performance. Complex models (deep neural networks, LLMs): high performance, low interpretability. There is no model that is simultaneously the most interpretable AND the most accurate — you must choose based on requirements.
Exam trigger "Regulated industry requires explainability for every decision" → choose simpler, interpretable model. "Maximum accuracy is the priority" → choose complex model. "Balance both" → use complex model + explainability tools (Clarify, Shapley values).
Measures of Trustworthiness & Truthfulness
Tools and methods to assess whether an AI is trustworthy: Subgroup analysis (does it work equally for all groups?), Label quality (is training data correctly labelled?), Human audits (manual review of outputs), SageMaker Clarify (automated bias + explainability), A2I (human-in-the-loop verification), Model Cards (documented limitations).
Exam trigger "How do you verify an AI model is trustworthy?" → combine multiple tools: automated (Clarify, Model Monitor) + human (A2I, audits) + documentation (Model Cards).

Data lifecycle: rules for how data moves from creation to deletion. Logging: record all data access and changes. Residency: data stays in specific geographic region (GDPR). Encryption: At Rest vs In Transit
At rest: data stored on disk — encrypted using AWS KMS. Even if storage is physically accessed, data is unreadable without the key. In transit: data moving between services — encrypted using TLS. AWS PrivateLink keeps traffic off the public internet entirely.
Exam trigger "Protect stored data" → encryption at rest + KMS. "Protect data moving between services" → encryption in transit + TLS. "No traffic over public internet" → PrivateLink.
ESOL note

At rest = data sleeping in a safe. In transit = data travelling in an armoured car. Both need protection: secure the safe AND secure the journey.

IAM — Identity & Access Management
Controls who can access which AWS resources and what actions they can perform. Key principle: least privilege — grant only the minimum permissions required for the job. Uses roles, policies, and permissions.
Exam trigger "Control who can call the Bedrock API" or "restrict data scientist from deleting S3 data" → IAM roles and policies with least privilege.
ESOL note

IAM = building security card system. Each person's card only opens doors they need. Least privilege = no extra doors. If a card is compromised, the damage is limited to only what that card could access.

Privacy-Enhancing Technologies (PETs)
Technical methods that protect personal data while still allowing AI to learn from it. Examples: Differential privacy (adds statistical noise to protect individuals), Federated learning (train on data without centralising it), Homomorphic encryption (compute on encrypted data without decrypting it).
Exam trigger "Train AI on sensitive data without exposing individuals" or "protect personal data during ML training" → Privacy-enhancing technologies (PETs).
ESOL note

PETs let AI learn from private data without seeing it directly. Like teaching someone statistics from a country's health data without ever knowing individual people's records. Differential privacy adds "noise" so no single person can be identified from the results.

Data Access Control & Data Integrity
Data access control: restricting who can read, modify, or delete training data and model outputs — using IAM, bucket policies, VPC endpoints. Data integrity: ensuring data has not been tampered with or corrupted — using checksums, versioning, audit trails.
Exam trigger "Prevent unauthorised modification of training data" → data integrity (checksums + CloudTrail). "Only ML team can access training dataset" → data access control (IAM + S3 bucket policy).
Compliance Frameworks
GDPR: EU data privacy — controls over personal data collection and processing. HIPAA: US health data protection — strict controls for PHI (Protected Health Information). SOC 2: security, availability, processing integrity audit standard. ISO 27001: international information security management standard. Algorithm accountability laws: emerging regulations requiring explainability for automated decisions.
Exam trigger "Healthcare AI must comply with…" → HIPAA. "AI serving EU customers must comply with…" → GDPR. "Download AWS compliance certificates" → AWS Artifact.
Source Citation & Data Origins
Responsible AI requires documenting where training data came from and how it was transformed: Data lineage (tracks data from source to model), Data cataloging (searchable inventory of all datasets), SageMaker Model Cards (document training data, intended use, limitations).
Exam trigger "Trace where the training data came from" → data lineage. "Find and inventory all available datasets" → data cataloging. "Document model limitations for regulators" → Model Cards.

Don't confuse withAmazon Forecast (Forecast = predicts future numerical values like demand or stock levels. Personalize = recommends items to specific users based on their behaviour. Both use ML but for different outputs).
Amazon Forecast
Managed time-series forecasting — predicts future values based on historical data patterns, automatically handling seasonality, trends, and external factors.
Use this whenThe company has historical time-stamped data and needs to predict future values — demand forecasting, inventory planning, financial projections, staffing levels.
Memory hookForecast = predict the future from the past. Time-series data IN, future numbers OUT.
Don't confuse withAmazon Personalize (Forecast = predicts future numbers from time-series data. Personalize = recommends items to individual users. Different inputs and outputs).
Amazon Fraud Detector
Managed ML service that detects fraudulent activities in real time — online transactions, account creation, fake reviews, payment fraud — using patterns learned from historical data.
Use this whenThe company needs to detect fraud or abuse in real time — suspicious transactions, fake account registrations, payment fraud — without building a custom ML fraud model.
Memory hookFraud Detector = alarm system for suspicious events. Transaction patterns IN, fraud probability OUT.
Don't confuse withAmazon Rekognition (Rekognition = visual analysis. Fraud Detector = transaction and event pattern analysis. Both detect "bad things" but in completely different domains).
Amazon Augmented AI (A2I)
Creates human review workflows for ML predictions — routes low-confidence or sensitive AI outputs to human reviewers before they take effect.
Use this whenThe company needs a human to review and approve AI decisions in high-stakes situations — medical diagnoses, content moderation, financial decisions — before the decision reaches the end user.
Memory hookAugmented = enhanced with extra help. AI does fast work; a HUMAN checks the uncertain cases.
Don't confuse withSageMaker Clarify (Clarify = automated bias analysis and explainability reports — no human reviewer. A2I = actual human beings review individual AI outputs in a workflow).
